## 快速测试指南

### 1. 测试审批流程

1. 使用 `sales_rep` 登录，申领公海客户
2. 使用 `team_leader` 登录，审批申领申请（团队层级）
3. 使用 `branch_manager` 登录，审批申领申请（分所层级）
4. 使用 `hq_admin` 登录，审批申领申请（总所层级）

### 2. 测试数据范围

1. 使用 `sales_rep` 登录，只能看到自己负责的客户
2. 使用 `team_leader` 登录，可以看到本团队所有客户
3. 使用 `branch_manager` 登录，可以看到本分所所有客户
4. 使用 `hq_admin` 登录，可以看到全所所有客户

### 3. 测试客户流转

1. 使用 `hq_admin` 或 `team_leader` 分配公海客户给 `sales_rep`
2. 使用 `sales_rep` 登录，跟进客户、记录拜访
3. 使用 `sales_rep` 确认合同交案
4. 使用 `sales_rep` 录入回款
5. 使用 `sales_rep` 录入律师费

### 4. 测试客户转交

1. 使用 `sales_rep` 登录，申请将客户转交给 `sales_rep2`
2. 使用 `team_leader` 登录，审批转交申请
3. 使用 `sales_rep2` 登录，查看转交过来的客户

## 注意事项

1. **密码统一**：所有测试账号密码均为 `123456`
2. **Mock 模式验证**：
   - 必须使用已存在的测试账号（用户名或手机号）
   - 密码验证：如果输入明文密码（如`123456`），会验证是否正确；如果前端自动RSA加密，Mock会跳过密码验证
3. **数据隔离**：不同角色看到的数据范围不同，请注意测试数据隔离
4. **审批链**：审批流程需要逐级审批，注意审批顺序
5. **权限控制**：部分功能按钮会根据角色权限显示/隐藏
