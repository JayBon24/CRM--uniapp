一、项目概述
1.1 项目背景
在法律服务行业竞争日趋激烈的背景下，广东万方律师事务所（以下简称“律所”）销售人员的客户管理效率、跟进质量及数据化运营能力直接影响律所业务拓展成效。当前传统客户管理模式存在客户线索分散、跟进流程不规范、客户状态标记主观、销售数据难以量化分析等问题，制约了销售团队的工作效率和业绩提升。为解决上述痛点，律所亟需开发一套专属的AI CRM系统，通过智能化手段实现客户全生命周期管理、销售流程标准化、数据驱动决策，助力销售团队精准触达客户、高效推进业务，提升律所整体市场竞争力。
1.2 项目目标
•	构建客户全生命周期管理体系，实现从公海客户、跟进客户、交案客户、回款客户到赢单客户的全流程可视化管理，确保客户资源不流失、跟进不中断。
•	引入AI智能能力，实现客户状态自动标记、日程提醒智能生成、销售话术精准推荐等功能，降低销售人员工作负担，提升跟进精准度。
•	建立完善的权限管理机制，清晰区分总所与分所管理人员、销售人员的权限，保障数据安全与管理规范。
•	搭建销售数据分析体系，通过多维度数据指标对比分析，生成可视化报告并提出整改建议，为销售管理决策提供数据支撑。
•	规范销售跟进流程，通过跟进频率提醒、客户线索回收机制等功能，提升销售跟进效率与客户转化效果。
1.3 适用范围
本系统适用于广东万方律师事务所总所及分所的销售人员、销售管理人员（总所级、分所级），核心服务于律所工程类法律业务相关客户的开发与维护全流程（聚焦工程款回收相关的调解、仲裁、诉讼等法律服务场景）。
二、核心功能设计
本系统核心功能围绕“客户管理全流程+AI智能赋能+权限管控+数据驱动”四大核心维度展开，覆盖客户生命周期各阶段、销售全流程及管理决策全场景，具体功能设计如下：
2.1 客户全生命周期管理模块
该模块是系统核心，实现客户从线索获取到赢单归档的全流程管理，按客户状态划分为五大核心子模块，各子模块功能相互衔接、数据实时同步。
2.1.1 公海客户管理
核心定位：客户线索池，实现线索集中管理、合理分配，保障线索资源高效流转。
•	线索管理：支持管理人员批量/单个下发客户线索，销售人员新增自有线索（需经管理人员审核后入库），所有线索信息统一录入，核心必填项含经办人（默认分配/申领人员），可选填引荐人；其他信息含客户基础信息、需求初步描述、来源渠道等。
•	线索编辑：支持销售人员/管理人员对线索信息进行修改完善，确保信息准确性。
•	客户等级标记：支持手动标记或通过AI初步判定客户等级（A：高价值、高意愿；B：高价值、有意愿；C：有价值、有意愿；D：无价值或无意愿），为线索分配提供依据。
•	线索分配：管理人员可根据销售人员能力、区域、客户类型等维度，将公海线索精准分配至对应销售人员；支持销售人员主动申领公海线索（需经管理人员审批）。
2.1.2 跟进客户管理
核心定位：已分配至销售人员的待跟进客户管理，聚焦跟进过程规范化、信息留存完整化。
•	客户信息管理：支持销售人员修改客户基础信息、需求详情等，新增客户信息核心规则——经办人（必填）：明确当前负责跟进该客户的销售人员，系统默认首次分配/申领线索的人员为经办人；
•	客户等级动态调整：支持手动或通过AI对话自动更新客户等级，根据跟进过程中获取的客户需求强度、付费能力等信息实时优化等级划分。
•	拜访足迹记录：销售人员需详细记录每次客户拜访信息，包括拜访时间、地点（支持地图定位标注）、参与人员（律所人员+客户方人员）、核心价值信息（如客户核心诉求、项目进展、竞争对手情况、合作意向程度等），洽谈持续的时间等。支持上传拜访照片、录音、文档等附件。
•	跟进计划制定：支持销售人员根据客户情况制定跟进计划，系统结合AI提醒功能保障计划落地。
•	客户转交功能：支持销售人员发起客户转交申请，填写转交原因、目标经办人等信息；需经所属管理人员审批通过后，完成客户归属转移，系统自动更新经办人信息，并同步该客户全生命周期数据（跟进记录、拜访足迹、客户等级等）至新经办人；转交过程全程留痕，生成转交日志供后续追溯。
2.1.3 交案客户管理
核心定位：已签订委托合同的客户管理，聚焦案件信息规范化录入与管理。
•	合同管理：支持销售人员上传委托合同扫描件/电子版（支持PDF、Word等格式），系统自动提取合同关键信息（如合同编号、客户名称、服务类型、服务费用、合同期限等）并生成电子档案。
•	案件信息生成：基于委托合同信息，自动生成案件基础信息（案件名称、案件类型、涉案金额、服务团队等），支持销售人员补充完善案件详情（如案件背景、核心争议点、推进节点计划等）。
•	案件状态同步：与律所案件管理系统对接（若有），实时同步案件推进状态，确保销售端与案件端信息一致。
2.1.4 回款客户管理
核心定位：已交案且通过调解、仲裁或诉讼方式收回工程款的客户管理，聚焦回款信息跟踪与记录。
•	回款信息记录：销售人员需录入回款相关信息，包括回款时间、回款金额、回款方式（调解回款、仲裁回款、诉讼回款）、回款凭证（支持上传）等。
•	催收类别关联：自动关联客户催收类别（仲裁、调解、诉讼），记录催收过程中的关键节点与成果，为后续客户维护提供参考。
•	案件收尾同步：同步案件回款后的收尾状态，确保客户服务闭环。
2.1.5 赢单客户管理
核心定位：已回款且客户已支付律师费用的客户管理，聚焦客户价值深挖与长期维护。
•	律师费用记录：录入客户律师费用支付信息（支付时间、金额、方式、凭证等），形成完整的费用台账。
•	客户价值归档：对赢单客户的全生命周期数据进行汇总归档，包括跟进记录、案件信息、回款信息等，为客户二次开发、转介绍提供数据支撑。
•	长期维护提醒：系统根据客户情况自动生成维护提醒（如节日问候、业务回访等），助力销售人员维护客户关系，挖掘潜在二次合作机会。
2.2 AI智能赋能模块
引入AI大模型能力，实现客户管理与销在售跟进的智能化升级，降低人工操作成本，提升工作效率与精准度。
2.2.1 客户状态AI自动标记
通过AI对话交互（支持文字、语音输入转文字），自动提取关键信息并完成以下标记，标记结果可手动修正，修正记录计入日志：
•	客户等级标记：基于对话中客户的需求强度、付费能力、合作意向等信息，自动判定为A/B/C/D级。
•	催收类别标记：根据客户案件涉及的工程款回收方式，自动标记为仲裁、调解、诉讼中的一种或多种（若涉及多种方式需区分主次）。
•	保全状态标记：通过对话信息判断客户案件是否需要财产保全，自动标记为“保全”或“非保全”；若为保全状态，需同步记录保全申请进度。
•	案件所辖法院标记：根据客户案件的地域、案件类型等信息，自动匹配并标记对应的管辖法院；若存在多个可能法院，需列出候选并提示销售人员确认。
•	案件律师记录：自动关联负责该客户案件的律师信息，支持销售人员补充或修改。
2.2.2 智能日历与提醒
•	日程提醒自动生成：基于客户跟进计划、拜访记录、案件关键节点（如开庭时间、调解时间、回款截止时间等），AI自动提取时间信息并写入系统日历，生成跟进提醒、节点提醒。
•	多维度提醒方式：支持系统内弹窗提醒、短信提醒、邮件提醒（销售人员可自行设置提醒方式与提前提醒时长）。
•	日程管理功能：支持销售人员手动添加、修改、删除日程，系统自动关联对应客户与案件信息，方便查看日程与客户/案件的关联关系。
2.2.3 跟进辅助智能推荐
•	跟进频率提醒：根据客户等级制定差异化跟进频率标准（如A级客户每周至少1次跟进，B级客户每两周至少1次跟进等），系统实时监控跟进进度，对未达标的销售人员发出提醒。
•	销售话术推荐：基于客户等级、需求类型（如工程款催收、合同纠纷等）、跟进阶段（初步接洽、需求深挖、方案推介等），AI自动推荐针对性的销售话术，助力销售人员高效沟通。
•	跟进频率提醒：根据客户等级制定差异化跟进频率标准（如A级客户每周至少1次跟进，B级客户每两周至少1次跟进等），系统实时监控跟进进度，对未达标的销售人员发出提醒。
•	销售话术推荐：基于客户等级、需求类型（如工程款催收、合同纠纷等）、跟进阶段（初步接洽、需求深挖、方案推介等），AI自动推荐针对性的销售话术，助力销售人员高效沟通。
•	展业技巧指导：针对跟进效果不佳的末尾客户（如多次跟进无进展、客户意向下降等），AI结合客户情况与行业案例，提供个性化的展业技巧突破战术（如调整沟通角度、补充行业案例、联合律师共同跟进等）。
2.2.4 语音指挥智能助理
系统AI对话框集成语音交互能力，支持销售人员通过语音指挥智能助理完成多种核心操作，提升操作效率，减少手动录入成本。语音交互采用“语音输入→AI意图识别→操作执行/确认→结果反馈”的全流程闭环，具体支持操作及实现逻辑如下：
•	客户状态语音标记：销售人员可通过语音指令（如“将XX客户等级标记为A级”“标记XX客户催收类别为仲裁”），智能助理自动识别客户信息与标记类型，完成状态更新；若存在信息歧义（如客户名称重复），将通过语音反馈询问确认。
•	客户信息语音增删改：支持语音指令新增客户基础信息（如“新增客户XX公司，经办人张三，引荐人李四”）、修改现有信息（如“将XX客户联系方式改为138XXXX1234”）、删除无效信息（如“删除XX客户的备用联系人信息”）；执行删除/关键信息修改操作前，智能助理将通过语音二次确认，避免误操作。
•	其他核心操作语音触发：可通过语音指令发起客户转交申请（如“申请将XX客户转交给王五，原因是地域负责调整”）、添加跟进记录（如“记录今日15点拜访XX客户，参与人员张三和李总，核心需求是工程款诉讼代理”）、生成日程提醒（如“添加3天后下午3点跟进XX客户的日程提醒”）等。
•	语音交互辅助：支持语音指令暂停/取消操作、查询操作结果（如“查询刚才标记的XX客户状态”）；智能助理具备多轮对话理解能力，可针对模糊指令进一步追问补充信息（如收到“标记客户等级”指令后，追问“请告知具体客户名称和目标等级”）。
2.3 权限管理模块
采用“角色-权限-数据”三级权限管控模式，清晰区分不同角色的操作权限与数据查看范围，保障系统数据安全与管理规范。
2.3.1 角色划分
•	总所管理人员：律所总部销售负责人，拥有系统最高管理权限。
•	分所管理人员：各分所销售负责人，拥有本所范围内的管理权限。
•	销售人员：一线销售执行人员，拥有个人负责客户的操作权限及部分公共数据查看权限。
2.3.2 权限明细
权限类型	总所管理人员	分所管理人员	销售人员
客户线索管理（公海）	新增、修改、删除、分配、审核全权限；查看全所公海线索	新增、修改、分配、审核本所公海线索；查看本所公海线索	申领公海线索、新增自有线索（需审核）；查看可申领的公海线索
客户信息管理（全生命周期）	查看、修改全所所有客户信息；删除无效客户信息	查看、修改本所所有客户信息；删除本所无效客户信息	查看、修改个人负责客户信息；不可删除客户信息
AI标记管理	查看、修正全所客户AI标记结果；配置AI标记规则	查看、修正本所客户AI标记结果；反馈AI标记问题	查看、修正个人负责客户AI标记结果；反馈AI标记问题

2.4 销售数据分析模块
基于系统内沉淀的销售全流程数据，构建多维度数据分析模型，实现销售业绩、工作效率、客户转化等方面的量化分析，为管理决策提供数据支撑。
2.4.1 核心分析指标
•	跟进频次：单个销售人员/团队在指定周期内的客户跟进总次数、平均跟进次数；不同客户等级的跟进频次分布。
•	新客户线索获取量：单个销售人员/团队在指定周期内新增的有效线索数量（经审核通过的线索）。
•	有效线索转化率：指定周期内，有效线索转化为跟进客户的比例；不同来源线索的转化率对比。
•	客户拜访频次：单个销售人员/团队在指定周期内的客户拜访总次数、平均拜访次数；重点客户（A/B级）拜访占比。
•	拜访成功率：指定周期内，拜访后客户意向提升或达成下一步合作意向的拜访次数占总拜访次数的比例。
•	单个客户平均洽谈时长：单个销售人员/团队在指定周期内，与客户每次洽谈的平均时长。
•	客户拜访周期：单个客户从首次拜访到达成合作（签约）的平均时间；不同客户等级的拜访周期对比。
•	客户转化漏斗：从公海线索→跟进客户→交案客户→回款客户→赢单客户的各环节转化率。
2.4.2 分析报告生成
•	多维度对比分析：支持按销售人员、分所/总所、时间周期（日/周/月/季度/年）等维度进行数据对比，清晰呈现个体与团队的工作差异。
•	可视化呈现：通过柱状图、折线图、饼图、漏斗图等多种图表形式展示分析结果，直观易懂。
•	自动生成报告：系统按预设周期（如每周、每月）自动生成销售数据分析报告，包含核心指标数据、趋势分析、差异对比等内容。
•	整改建议输出：AI基于数据分析结果，针对表现不佳的指标（如某销售人员有效线索转化率过低、重点客户拜访占比不足等），结合行业最佳实践与律所实际情况，自动提出针对性的整改建议（如加强线索筛选培训、优化重点客户跟进计划等）。
2.4.3 数据导出与分享
支持将分析报告、原始数据导出为Excel、PDF等格式，方便管理人员存档、汇报；支持在系统内分享报告给指定角色，便于团队协同沟通。
2.5 客户线索回收机制
为保障客户线索资源的高效利用，避免资源浪费，系统设置自动回收规则，对符合以下条件的客户线索自动回收至公海：
•	销售人员领取线索后，1个月内未形成有效拜访（有效拜访定义：完成规范的拜访记录录入，且获取到有价值的客户信息）。
•	客户从跟进状态开始，6个月内未签订委托合同（未进入交案客户状态）。
•	从客户上一单案件赢单开始，6个月内未签订新的委托合同。
