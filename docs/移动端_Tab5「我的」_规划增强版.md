# Tab5「我的」规划增强版（基于界面规划 + 当前代码现状）

> 目标：把「我的」做成 **个人中心 + 设置 +（管理角色）审批/管理入口** 的统一入口，且“销售端尽量干净，管理入口折叠在我的-管理”。

---

## 1. 角色与数据范围（沿用 Tab1 口径）

### 1.1 角色层级（RoleLevel）
- **总所管理（HQ）**：仅管理职能（无销售职能）
- **分所管理（BRANCH）**：仅管理职能（无销售职能）
- **团队（TEAM）**：基层管理 + 也具备销售职能（既能看团队内销售人员，也会作为经办人推进自己的客户）
- **销售（SALES）**：仅销售职能

### 1.2 数据范围（OrgScope）
- **HQ**：全所
- **BRANCH**：本分所
- **TEAM**：本团队
- **SELF**：本人

> 说明：移动端 Tab5 的“入口可见性”与“审批/管理列表可见数据范围”都按上述规则计算。

### 1.3 AI 接入与扩展点（统一要求）
> Tab5 涉及审批/反馈/设置等能力，均需预留 AI 扩展点（AI 总结、风险提示、审批建议草案）与可追溯字段，统一规范见：`docs/AI接入与扩展点规范.md`。

## 2. 实体与字段定义（Tab5 领域）

### 2.1 CurrentUser（当前用户）
来自 `userStore`（或后端 `/user/info`）：
- `id`：用户ID
- `name/nickName`：展示名
- `avatar`：头像
- `roleLevel`：HQ/BRANCH/TEAM/SALES
- `orgScope`：HQ/BRANCH/TEAM/SELF
- `teamId/teamName?`
- `branchId/branchName?`
- `hqId/hqName?`

### 2.2 ReminderPreference（提醒偏好）
用于 Tab1 的计划/跟进提醒（一期仅保留提前量，渠道选择隐藏）
- `default_remind_advance_minutes`：15 / 30

### 2.3 PrivacyAuthorization（隐私授权状态）
只做展示与跳转系统设置：
- `location`：已授权/拒绝/未询问
- `camera`：已授权/拒绝/未询问
- `album`：已授权/拒绝/未询问
- `microphone`：已授权/拒绝/未询问（若未来做语音）

### 2.4 FeedbackTicket（反馈工单）
一期可 mock + 本地存储：
- `id`
- `type`：bug / suggestion / ai_tag_feedback
- `content`
- `images[]`：可选
- `contact`：手机号/微信（可选）
- `context`：（可选）`{ client_id?: string; field?: string }`
  - 从「我的」进入：为空
  - 从客户详情进入：自动带上（例如 `client_id=xxx`，`field` 选填）
- `status`：open / processing / resolved
- `created_at`

### 2.5 ApprovalTask（审批任务）（管理角色专属）
一期只做列表+详情+通过/驳回（写原因），接口后续对接 Tab1 流程：
- `id`
- `type`：
  - `LEAD_CLAIM`（线索申领）
  - `LEAD_CREATE`（销售新增线索审核）
  - `CLIENT_TRANSFER`（客户转交审批）
- `applicant_user_id/name`
- `client_id/client_name`
- `payload`：申请内容（按 type 区分结构，必须强类型）
  - `LeadClaimPayload`：`{ client_id, name, reason }`
  - `LeadCreatePayload`：包含“新建线索表单所有字段”
  - `ClientTransferPayload`：`{ client_id, from_user, to_user, reason }`
- `status`：pending / approved / rejected
- `approval_chain`：层层审批链（默认：TEAM→BRANCH→HQ）
- `current_approver_role`：当前需要审批的层级
- `history[]`：审批历史（每一层的通过/驳回/时间/原因）
- `created_at`
- `approved_at`
- `approval_reason`

## 3. 页面信息架构（IA）

### 3.1 P5-1 个人中心首页（所有人可见）
**展示区块**
- 个人信息卡：头像、姓名、角色/所属分所/团队
- 快捷入口（通用）：个人资料、设置、反馈与支持、关于
- 管理入口折叠区（仅管理角色可见）：审批中心、公海分配、团队日程、管理报表入口（跳 Tab4）、规则配置

**交互**
- 点击头像/资料：进入「个人资料」页（一期最小：头像/姓名/手机号展示）
- 点击设置：进入「系统设置」
- 点击反馈：进入「反馈与支持」
- 点击关于：进入「关于」
- 点击管理入口：跳转至相应管理子页面或占位说明页

### 3.2 P5-2 我的设置（所有人可见）
**设置项**
- 提醒偏好：仅提前量（15分钟/30分钟）
- 用户协议与隐私政策入口
- 缓存清理
- 退出登录（清空 token/store → 跳转登录）
- 开发者工具（仅 Mock 环境）：角色切换（已实现）

### 3.3 P5-3 我的数据（个人维度）（所有人可见，一期可占位）
- 展示个人指标汇总（可复用 Tab4 指标组件/或先占位）

### 3.4 P5-4 反馈与支持（所有人可见）
- 提交反馈（类型 + 文本 + 图片 + 联系方式）
- AI 标记反馈（可选：绑定到客户/字段）
- 查看历史反馈（一期可选）

### 3.5 P5-M1 审批中心（管理角色）
**列表页**
- Tab：待审批 / 已处理
- 列表项：类型、申请人、客户、时间
**详情页**
- 展示申请内容与附件
- 底部两个按钮：通过 / 驳回
- 驳回：弹窗输入 `reject_reason`（必填）

> 与 Tab1 流程对齐：申领公海、销售新增线索、客户转交，都走审批中心。

### 3.6 P5-M2 公海分配（管理角色）
**核心功能**
- **公海列表查询**：展示当前池内所有待分配线索（支持按地区、等级筛选）。
- **批量/单个下发**：勾选线索后，选择目标销售人员进行分配。
- **自动分配日志**：查看系统自动分配的历史记录（可选）。
- **分配建议**：AI 根据销售人员的地域匹配度、当前负载推荐分配人（二期预留）。

### 3.7 P5-M3 团队日程（管理角色）
**核心功能**
- **全员日程视图**：以周/月日历形式查看本团队/本所所有销售的拜访与跟进计划。
- **人员钻取**：点击特定销售，查看其个人的详细待办列表。
- **冲突提醒**：识别同一时间段内，同一客户被多人计划拜访的冲突情况。
- **工作饱和度统计**：直观展示谁的日程排满了，谁还有空档。

### 3.8 P5-M4 管理报表入口（管理角色）
**核心功能**
- **快捷跳转**：点击后直接跳转至 Tab4「数据报表」页面。
- **自动切换维度**：根据当前管理角色级别（分所/团队），自动在报表页应用对应的统计维度和数据范围。

### 3.9 P5-M5 规则配置（管理角色）
**核心功能**
- **回收时限设置**：配置不同等级客户（A/B/C/D）在无有效跟进/无拜访情况下的自动收回公海天数。
- **跟进频率标准**：设置各阶段客户的最小跟进频率（如：面谈阶段客户需 3 天内有跟进）。
- **回收预警开关**：开启/关闭临近回收时的系统强提醒功能。
- **销售阶段转化阈值**：定义达到什么条件（如：必须有附件）才允许手动切换到下一阶段。
